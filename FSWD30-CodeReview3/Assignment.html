CodeReview 3

Good Morning Students!

Welcome to your 3rd CodeReview!  Now when you have some solid knowledge in HTML, CSS and Javascript, we are even more excited to see what next big thing that you will develop!

You will need to achieve 60 points to pass this CodeReview successfully. Partially-accomplished criteria will be judged according to the achievement. Based on your achievements, actual grading (green / yellow / red ) will be further created by our staff.

You can submit your solution until Sunday 10.DEC.2017, 17:00:00.


Project description:

Car insurances are one of the big hassles of adult life. Even more so, finding an insurance-company nearby and getting a quote can be surprisingly tedious! This weeks code-review example is going to solve all that!


Your tasks will be:

1. to create a homepage for a car-insurance company.

Create a landing-page with a rich design, which displays at least the company name and all its employees.

To display the employees with their name and email address, create a two-dimensional array that stores these values such as:

var employees = [
 ["Bill", "bill@example.com"], 
 ["Jill", "jill@example.com"] 
];
Note: if you prefer to use object notation with objects in the array (as in JSON), this is fine as well. Both approaches are acceptable.


Loop through the array and display this data on the page in a fitting manner. Remember that you can place entire HTML code-snippets mixed with variables in another variable, for example like this:

var name = 'John';
var snippet = "<div class='your_class'><p>" + name + "</p></div>";

2. Create a second page, with an input form where a customer can ask for an insurance-quote.

Create a form that takes the customers name, age, country of origin (Germany, Austria and Italy) and horsepower of their car. Ensure the correctness of datatypes which are written in the form (Strings and Numbers).

Upon clicking the "calculate" button of the form, run a named JavaScript function that does the following:

  • For the customers from Italy there is 10% discount.
  • For a cars with horsepower greater than 150 add additional cost from 80 Euros.
  • Display the offer on the same page, by providing detailed information of the calculation


For this CodeReview, the following criteria will be graded:

  • (10) Creation of GitHub project (Repository name: firstname-lastname-FSWD30-CodeReview2), successful “push” to the repository and correct information to CodeFactory through the learning management system (https://lms.codefactory.wien/) with GitHub link for clone procedure.

  • (10) Correct usage of HTML structure (proper order of HTML elements)

  • (10) Correct implementation of the navigation (logo placement, web page name and the navbar)

  • (10) Correct code structure (aka code indention) that makes it easier for humans to read

  • (25) Use CSS selectors (classes and/or IDs) to style your web page ( the colors, font size, font style etc.)

  • (15) Correct CSS styling according to the provided design (colors, layout, logo etc. Pay attention to the small details like shadow, rounded corners etc.)

  • (10) Store the city data into Javascript variables (city name, country and average price). Display and place all those variables of the related city using getElementById() method

  • (5) Calculate the sum of the prices for every city and store the average price in a new variable. Use those variables to change the average price on the screen by using getElementById() method

  • (5) Calculate the difference of the average prices of London and Amsterdam and use getElementById() to display the difference into the rounded blue rectangle


Bonus Points:

  • (10) Your page styles is using width properly to be a bit responsive (it looks fine with different web browser size)

  • (10) Use a function to calculate the average prices


Please note: hard-coded output from Javascript will be acceptable and positively graded (we didn't cover so far conditions and loops). If you are familiar with those concepts feel free to use them.
